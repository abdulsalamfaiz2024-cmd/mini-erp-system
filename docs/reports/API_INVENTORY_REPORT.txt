CRITICAL WORKFLOW APIs
═══════════════════════════════════════════════════════════════════

These APIs drive the document state machine:

1. ApprovalService.approve_document
   - Central approval mechanism
   - Transitions documents through states
   - Enforces permission levels
   - Optimistic locking via version control
   
2. ApprovalService.reject_document
   - Sends documents to Rejected state
   - Requires approval_level >= 3

3. WorkflowService.initialize_document_state
   - Creates initial workflow tracking
   - Called by all document creation methods

4. SalesOrderService.* (7 methods)
   - Wrapper layer enforcing sales order workflow
   - Each method maps to specific state transitions

HIGH-RISK APIs (Database Mutations)
═══════════════════════════════════════════════════════════════════

SALES FLOW:
  1. SalesManager.create_order
     → INSERT sales_orders, sales_order_items
     → UPDATE customers.current_balance
     → inventory_manager.deduct_stock (FIFO consumption)
     → workflow_service.initialize_document_state
     
  2. SalesManager.generate_invoice
     → UPDATE sales_orders.status = 'invoiced'
     → UPDATE customers.current_balance += total
     
  3. SalesManager.record_payment
     → INSERT payments
     → UPDATE customers.current_balance -= amount

INVENTORY FLOW:
  1. InventoryManager.add_stock
     → INSERT inventory (FIFO batch)
     → UPDATE products.cost_price
     
  2. InventoryManager.deduct_stock
     → UPDATE inventory quantities (FIFO)
     → Can raise ValueError if insufficient

PURCHASING FLOW:
  1. PurchasingManager.create_purchase_order
     → INSERT purchase_orders, purchase_order_items
     
  2. PurchasingManager.receive_items
     → inventory_manager.add_stock (multiple)
     → UPDATE suppliers.current_balance
     
  3. PurchasingManager.record_payment
     → UPDATE suppliers.current_balance

EXCEPTION TYPES ACROSS ALL APIs
═══════════════════════════════════════════════════════════════════

ValidationError:
  - Raised by: sales, employee, finance, inventory, purchasing
  - Indicates: Business rule violations, insufficient stock, etc.
  
WorkflowError:
  - Raised by: workflow service, ApprovalService
  - Indicates: Invalid state transitions, missing workflow
  
ConcurrentModificationError:
  - Raised by: ApprovalService.approve_document
  - Indicates: Optimistic locking failure
  
DatabaseError:
  - Raised by: Most write operations
  - Indicates: SQL constraint violations, connection issues
  
IOError:
  - Raised by: Reporting exports, batch operations
  - Indicates: File system access failures
