# Plan 2.4: Finance, Reporting & Ledger Automation (Exhaustive Detail)

## 1. Scope & Objective
This document defines the automated accounting engine. It ensures that every business event (Selling, Buying, Returning) is accurately recorded in the General Ledger for financial reporting.

---

## 2. Automated Ledger Entry (ALE) Specification
The system must never allow a manual "ledger entry" for standard transactions. They must be generated by code to prevent human error.

### Event: Invoice Generated
- **Debit**: `Accounts Receivable` (Customer Asset)
- **Credit**: `Sales Revenue` (Income)

### Event: Expense Approved
- **Debit**: `Expense Account` (Administrative/Operational)
- **Credit**: `Cash/Bank` (Current Asset)

---

## 3. Reporting Engine (`services/report_service.py`)
This service aggregates multi-table data into the format required by the UI.

- **Finance Dashboard**: Calculates "Net Profit" (Revenue - Expenses) on the fly.
- **Aging Report**: Grouping unpaid invoices by `<30`, `31-60`, `61-90` days.
- **Tax Report**: Summating `total_tax` across all invoices and expenses for the period.

---

## 4. Key Functions & Locations
| Function Name | Location | Description |
| :--- | :--- | :--- |
| `generate_fin_summary`| `controllers/finance_controller.py` | Powers the Finance Dashboard charts. |

## 5. Verification Steps
1. [ ] **Double-Entry Balance**: Verify that for every transaction, `Sum(Debits) == Sum(Credits)`.
2. [ ] **Reporting Accuracy**: Create 5 invoices totaling $500; verify that the Finance Dashboard shows $500 Revenue.
3. [ ] **Journal Audit**: Verify that every Journal Entry can be traced back to a specific `Invoice_ID` or `Expense_ID`.
4. [ ] **Export Integrity**: Verify that data requested for "Financial Reports" matches the UI mappings in `financial_reports_spec.md`.
